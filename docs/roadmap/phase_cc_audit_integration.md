# Phase CC: Claude Code Audit Integration

**Phase Name:** The Auditor  
**Status:** Ready for implementation  
**Decision numbers:** #103–#107  
**Prerequisite PRs:** #130 (reasoning_trace + max_symbols — merged)

---

## Overview

Integrate AETHER with Claude Code via MCP to create a structurally-informed
code auditing workflow. AETHER provides graph centrality, health scores,
dependency cycles, community boundaries, and reasoning traces. Claude Code
provides Opus-level semantic understanding. Together they enable targeted,
persistent code auditing that neither can do alone.

### Cost model

| Pipeline | Cost/5K symbols |
|---|---|
| Current (scan → triage → deep via API) | ~$51 |
| Hybrid (scan → triage → Claude Code deep) | ~$8 + subscription time |

## Decision Register

### Decision #103: Slash commands generated by init-agent

**Context:** Users need a zero-friction entry point for auditing and refactoring
in Claude Code. Claude Code supports custom slash commands via `.claude/commands/`
markdown files.

**Decision:** Extend `init-agent` to generate three slash commands alongside
existing CLAUDE.md, skill, codex-instructions, and cursor-rules files:
- `/audit <crate-or-file>` — deep bug audit using AETHER structural intelligence
- `/refactor <crate-or-file>` — find refactoring opportunities using health + community data
- `/audit-report` — show all audit findings from previous sessions

**Rationale:** Slash commands are discoverable (`/help` lists them), repeatable,
version-controllable, and require zero manual setup after `init-agent`. They
degrade gracefully — instructions include fallbacks for tools that aren't available yet.

### Decision #104: sir_audit table (schema v18)

**Context:** Audit findings need to be queryable by severity, category, status,
and symbol. Free-text project notes via `aether_remember` work but don't support
structured filtering.

**Decision:** Add dedicated `sir_audit` table with columns for symbol_id,
audit_type, severity, category, certainty, trigger_condition, impact,
description, related_symbols, model, provider, reasoning, status, timestamps.
Schema version bumps from 17 to 18.

### Decision #105: Audit + SIR inject MCP tools

**Context:** Claude Code needs MCP tools to submit, query, and resolve audit
findings without falling back to CLI or free-text notes. Additionally,
`sir_inject` (writing improved SIR back to the store) is currently CLI-only,
forcing Claude Code to construct bash commands with JSON escaping.

**Decision:** Add five new MCP tools:
- `aether_audit_submit` — submit a structured audit finding
- `aether_audit_report` — query findings by crate/severity/status/category
- `aether_audit_resolve` — mark a finding as fixed/wontfix/confirmed
- `aether_sir_inject` — write improved SIR annotation back to store via MCP
  (reimplements core persist logic using store methods directly, skips embedding refresh)
- `aether_audit_candidates` — ranked list of symbols most in need of deep review

### Decision #106: CLAUDE.md audit section

**Context:** The generated CLAUDE.md needs an audit workflow section so Claude
Code automatically knows how to use AETHER tools for auditing.

**Decision:** Add an "Audit Workflow" section to the CLAUDE.md template covering
target finding, deep analysis process, bug categories, and result recording.

### Decision #107: aether_sir_context MCP tool

**Context:** Claude Code currently needs 3-5 separate MCP calls to assemble
context for one symbol (get_sir + dependencies + read source + health). The CLI
`sir-context` command (4018 lines) does this in one call with token budgeting,
but it's not accessible from the MCP binary.

**Decision:** Add `aether_sir_context` MCP tool as a simplified reimplementation
covering 6 of the CLI's 9 layers (source, SIR, graph, health, reasoning, tests).
Configurable token budget and layer selection. Omits coupling, memory, and
broader_graph layers which can be queried separately.

3. Extends `files_for_platform()` to emit `.claude/commands/audit.md`, `.claude/commands/refactor.md`, `.claude/commands/audit-report.md` for Claude platform
4. Adds audit workflow section to CLAUDE.md template

- Include fallback instructions: "If `aether_audit_candidates` is not available, use `aether_health` instead"

### Stage CC.2: sir_audit table + audit MCP tools + aether_sir_inject

**Type:** Medium PR (~600 lines across store + MCP)  
**Scope:** `crates/aether-store/`, `crates/aether-mcp/src/tools/`, `crates/aetherd/`  
**Schema change:** v17 → v18

**What it does:**
1. Adds `sir_audit` table in schema v18 migration
2. Adds store methods: `insert_audit_finding`, `query_audit_findings`, `resolve_audit_finding`
3. Adds `tools/audit.rs` with four MCP tools: `aether_audit_submit`, `aether_audit_report`, `aether_audit_resolve`, `aether_sir_inject`
4. `aether_sir_inject` closes the feedback loop — Claude Code can write improved SIR back to the store via MCP instead of bash
5. Registers tools in `router.rs`
6. Updates `check_compatibility("core", 18)` in all 5 call sites
7. Adds CLI command: `aetherd audit-report [--crate X] [--min-severity Y] [--status Z]`

**aether_sir_inject design notes:**
- Reimplements core persist logic using store methods directly (write_sir_blob, upsert_sir_meta, record_sir_version_if_changed)
- Cannot use SirPipeline (lives in aetherd binary crate, not a library)
- Uses aether_sir::canonicalize_sir_json and sir_hash from the library crate
- Intentionally skips embedding refresh — adds note to response
- Confidence blocking: rejects overwrite if existing confidence > 0.5 unless force=true

**check_compatibility sites (all must update 17→18):**
- `crates/aether-mcp/src/state.rs` lines 132, 162, 193, 224
- `crates/aether-dashboard/src/state.rs` line 42

**sir_audit table schema:**
```sql
CREATE TABLE sir_audit (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    symbol_id TEXT NOT NULL,
    audit_type TEXT NOT NULL,
    severity TEXT NOT NULL,
    category TEXT NOT NULL,
    certainty TEXT NOT NULL,
    trigger_condition TEXT NOT NULL,
    impact TEXT NOT NULL,
    description TEXT NOT NULL,
    related_symbols TEXT DEFAULT '[]',
    model TEXT NOT NULL,
    provider TEXT NOT NULL,
    reasoning TEXT,
    status TEXT NOT NULL DEFAULT 'open',
    created_at INTEGER NOT NULL,
    resolved_at INTEGER,
    FOREIGN KEY (symbol_id) REFERENCES sir(id) ON DELETE CASCADE
);
CREATE INDEX idx_audit_symbol ON sir_audit(symbol_id);
CREATE INDEX idx_audit_severity ON sir_audit(severity);
CREATE INDEX idx_audit_status ON sir_audit(status);
CREATE INDEX idx_audit_category ON sir_audit(category);
```

### Stage CC.2b: aether_sir_context MCP tool

**Type:** Medium PR (~400 lines)  
**Scope:** `crates/aether-mcp/src/tools/`  
**Schema change:** None

**What it does:**
1. Adds `tools/context.rs` with `aether_sir_context` MCP tool
2. Assembles token-budgeted context for a symbol in one call, replacing 3-5
   separate tool calls
3. Covers 6 layers: source code, SIR annotation, graph neighbors, health
   signals, reasoning trace, test intents
4. Supports markdown (default) and JSON output formats
5. Configurable token budget (default 8000) and layer selection

**Design notes:**
- Simplified reimplementation of CLI sir-context (4018 lines) using store
  methods accessible from the MCP crate
- Omits coupling, memory, and broader_graph layers (available via separate tools)
- Token budgeting: source 40%, graph 20%, SIR 15%, reasoning 10%, tests 10%, health 5%

### Stage CC.3: aether_audit_candidates MCP tool

**Type:** Small PR (~250 lines)  
**Scope:** `crates/aether-mcp/src/tools/audit.rs` (extend)  
**Prerequisite:** Reasoning trace data populated (run triage with thinking)

**What it does:**
1. Adds `aether_audit_candidates` MCP tool to existing `audit.rs`
2. Combines health report data (pagerank, betweenness, test_count) with SIR metadata (confidence, generation_pass) and reasoning_trace uncertainty signals
3. Returns ranked list of symbols most in need of deep review

**Input:** top_n, optional crate/file scope, min_risk, include_reasoning_hints  
**Output:** ranked candidates with risk_score, risk_factors, current_confidence, generation_pass, reasoning_hint, thinking_priority

### Stage CC.4: aether_audit_cross_symbol MCP tool (future)

**Type:** Medium PR  
**Prerequisite:** CC.3  

Traverses dependency graph from a root symbol, collects SIR + source + reasoning
for the full call chain, assembles structured context for cross-boundary analysis.
Deferred — build after validating CC.1–CC.3 in real audit sessions.
