## Overview

Phase CC integrated AETHER with Claude Code via MCP to create a
structurally-informed code auditing workflow. AETHER provides graph
centrality, health scores, dependency cycles, community boundaries,
and reasoning traces. Claude Code provides Opus-level semantic
understanding. Together they found 42 real bugs across two crates
in a single session.

## Pending Fixes (from audit findings)

### Fix 1: Reasoning trace not persisting (~90% of triage results lost)
- **Root cause:** `persist_sir_payload_into_sqlite` conditionally skips
  `upsert_sir_meta` when SIR hash unchanged, losing generation_pass and
  reasoning_trace for symbols where triage produces identical SIR to scan.
- **Impact:** Batch triage costs money but results silently discarded.
- **Fix:** Make metadata upsert unconditional. Belt-and-suspenders: ingest
  post-persist upsert explicitly carries payload values.
- **Prompt:** `fix_reasoning_trace_persist_prompt.md`

### Fix 2: Non-atomic multi-step writes (findings #1, #2, #7, #25, #27)
- **Root cause:** 3+ functions perform multi-step SQLite writes without
  transaction wrappers. Crash mid-sequence leaves orphaned data.
- **Fix:** Create unified `delete_symbol_fully()` covering all 16 tables.
  Wrap all multi-step writes in `Transaction::new_unchecked`.
- **Prompt:** `fix_transaction_audit_prompt.md`

### Fix 3: Silent error suppression (findings #6, #13, #23, #26 + ~8 more)
- **Root cause:** `.ok()` on SIR reads, `if let Ok(lock)` on mutexes,
  `unwrap_or(0)` on timestamps — all suppress errors silently.
- **Fix:** Add `tracing::warn!` to all instances. Preserve behavior but
  make failures observable.
- **Prompt:** `fix_silent_error_suppression_prompt.md`

## Validation Test Results

### Known bug-finding symbols
Not formally tested against the 8 benchmark symbols from the spec.
The audit workflow found 42 real bugs independently — validating the
approach without needing the pre-selected test cases.

### Trait dispatch graph limitation
Confirmed: `aether_audit_cross_symbol` cannot resolve callers through
trait dispatch indirection. Rust trait dispatch creates edges to the
trait method, not the concrete implementation. Documented as a known
limitation.

### Reasoning trace characteristics
- Low thinking: ~9% trace rate, 800-2000 chars, model chooses when to think
- Medium thinking: 100% trace rate, 1700-2900 chars, guaranteed thinking
- Model misinterprets "SIR" acronym — fixed by adding "(Semantic Intent Record)"
  to prompts. Applied to all 4 prompt entry points.
