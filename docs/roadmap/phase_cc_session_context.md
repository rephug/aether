## Key source inspection findings

1. **`aether_sir_inject` is CLI-only** (in `aetherd/src/sir_inject.rs`), NOT an
   MCP tool. Being added as MCP tool in CC.2. Core logic: resolve symbol, read
   existing SIR, merge fields, persist via store methods, refresh embeddings.
   MCP version skips embeddings (SirPipeline is in aetherd, not a library crate).
   Uses `aether_sir::canonicalize_sir_json` and `sir_hash` for SIR serialization.
   Store methods: `write_sir_blob`, `upsert_sir_meta`, `record_sir_version_if_changed`.

2. **`aether_sir_context` is CLI-only** (in `aetherd/src/sir_context.rs`), 4018
   lines with all functions `pub(crate)`. Being added as simplified MCP tool in
   CC.2b covering 6 of 9 layers (source, SIR, graph, health, reasoning, tests).

3. **MCP god file is already refactored.** The crate is 8206 lines across 15
   well-organized files in `tools/`. Memory note about "4799 lines" is stale.
   Adding `tools/audit.rs` and `tools/context.rs` follows the existing pattern.

4. **`TOOL_DESCRIPTIONS` in templates/mod.rs is outdated** — lists only 10 tools
   but router.rs has 26 registered MCP tools. CC.1 prompt includes updating this.

5. **Schema v17** added `reasoning_trace` column to `sir` table (PR #130).
   v18 will add `sir_audit` table.

6. **`check_compatibility("core", N)` has 5 call sites:**
   - `aether-mcp/src/state.rs` lines 132, 162, 193, 224
   - `aether-dashboard/src/state.rs` line 42

7. **PR #130 also added `max_symbols`** on batch — not a separate PR as memory
   stated.

8. **Contract tools (add/list/remove/check)** exist as CLI commands but have
   no MCP exposure. Deferred to CC.5 — natural follow-on for "audit → fix →
   contract" workflow.

9. **Gap analysis summary — CLI commands with no MCP equivalent:**
   - `sir-inject` → CC.2 (critical)
   - `sir-context` → CC.2b (high value)
   - `contract *` → CC.5 (future)
   - `sir-diff` → covered by `aether_why_changed`
   - `communities` → covered by `aether_health`
   - `coupling-report` → covered by `aether_blast_radius` + `aether_drift_report`
   - `seismograph/task-*/fsck/regenerate/batch/continuous` → operational, not needed in MCP

## Known issues that may affect implementation

- **Readonly database bug:** `compute_health_report()` opens readonly but
  downstream writes. May affect `aether_health` in some paths. CC.3 depends
  on health data — if it hits this, open read-write.

- **SurrealKV lock contention:** Dashboard and CLI can't share workspace.
  `pkill -f aetherd` before running CLI. Claude Code MCP has same constraint.
